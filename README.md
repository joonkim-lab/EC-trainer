# 영작 트레이너 (English Sentence-Writing Trainer)

한국형 학습자가 **"뼈대 → 확장"** 방식으로 영어 문장을 직접 만들며
영어 자신감 → 스피킹 자신감으로 이어지도록 설계한 컴퓨터 기반 훈련 프로그램입니다.

> 교재: 유은하 『영어로 문장 만들기 훈련』 **1·2·3차 임계점** (사람in)
> → 1차 = **레벨 1**, 2차 = **레벨 2**, 3차 = **레벨 3**

---

## 무엇을 하는 프로그램인가

- **로그인 후 배치 테스트**로 아이마다 시작 레벨(1/2/3)을 정합니다.
- 교재 **목차 순서**대로 진행하며, 각 단원 시작에 *"이 문장 만들기는 무엇을 위한 것인지"* 짧은 설명을 보여준 뒤 연습합니다. (빈칸 채우기 방식은 쓰지 않습니다.)
- AI가 **한국어로, 한 번에 한 가지만**, 따뜻하지만 정확하게 피드백합니다. 의미 → 어순 → 뼈대 우선순위로 봅니다.
- 유닛 통과(기본 75%) → **레벨업 시험**(기본 85%) 통과 → 다음 레벨로 승급.
- **묵상집형 스케줄러**: 누가 오늘 연습했는지 ✅/⬜ 로 서로 보고 👏 응원 → 함께 독려.
- 레벨 3 완료 → **듀오링고 시험 → AP 준비**로 이어지는 다음 단계 안내.

향후: 우리 아이들 데이터를 바탕으로 학교의 한국형 학습자들에게 확장 적용.

---

## 폴더 안에 무엇이 들어있나

| 파일 | 역할 | 내가 건드릴 곳 |
|---|---|---|
| **index.html** | 앱 전체(화면·로직). 더블클릭하면 브라우저에서 바로 미리보기 가능 | 보통 그대로 |
| **curriculum.js** | 3개 레벨 전체 커리큘럼(교재 목차 기반) + 통과 기준 상수(RULES) | 단원/기준 조정 시 |
| **firebase-config.js** | Firebase 연결 값 + AI 함수 주소(FUNCTION_URL) | ★ 콘솔 값으로 채움 |
| **firestore.rules** | 보안 규칙(허용 이메일 명단) | ★ 이메일 4개 교체 |
| **functions/index.js** | AI 채점·출제 중계 함수(API 키 안전 보관) | 모델 변경 시 |
| **functions/package.json** | 함수 의존성 | 그대로 |
| **firebase.json** | 호스팅·규칙·함수 배포 설정 | 그대로 |
| **SETUP_Firebase.md** | ① Firebase 설정 차근차근 가이드 | 따라 읽기 |
| **SETUP_Domain.md** | ② 나만의 도메인 연결 가이드 | 필요 시 |
| **README.md** | 이 문서 | — |

> ★ 표시 두 곳(`firebase-config.js`, `firestore.rules`)만 내 값으로 채우면 됩니다. 나머지는 거의 그대로.

## 시작하는 순서 (요약)

1. **그냥 구경만:** `index.html` 을 더블클릭 → 로그인·저장·AI 없이 화면/흐름만 미리보기.
2. **실제로 쓰기:** **SETUP_Firebase.md** 를 위에서부터 따라 하기.
   - 프로젝트 만들기 → config 채우기 → 구글 로그인 켜기 → Firestore + **이메일 4개** 넣기 → API 키 시크릿 저장 → 함수 배포(→ FUNCTION_URL 받기) → `firebase deploy`.
3. **주소 예쁘게:** (선택) **SETUP_Domain.md** 로 나만의 도메인 연결.

---

## 승급 기준(조정 가능)

`curriculum.js` 맨 아래 `RULES` 에서 숫자만 바꾸면 난이도가 조정됩니다.

| 항목 | 기본값 | 의미 |
|---|---|---|
| `PLACEMENT_SIZE` | 6 | 배치 테스트 문항 수 |
| `UNIT_SET_SIZE` | 8 | 한 유닛 연습 문항 수 |
| `UNIT_PASS_RATIO` | 0.75 | 유닛 통과 기준(75%) |
| `LEVELUP_SET_SIZE` | 12 | 레벨업 시험 문항 수 |
| `LEVELUP_PASS_RATIO` | 0.85 | 레벨업 통과 기준(85%) |
| `PASS_SCORE` | 70 | 문장 하나가 '통과'로 인정되는 점수 |

## 설계 원칙 (교육적 약속)

- **뼈대 먼저:** 주어+동사("누가 한다")를 세우고 바깥으로 확장. 무생물 주어는 오해 없이 신중히 다룹니다.
- **한국어 피드백, 한 번에 하나:** 한꺼번에 여러 지적을 쏟지 않습니다. 따뜻하되 정확하게.
- **빈칸 채우기 금지:** 학생이 **문장 전체를 스스로** 만들도록 합니다.
- **설명 후 연습:** 각 단원이 "무엇을 위한 것인지" 먼저 이해하고 시작.
