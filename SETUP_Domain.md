# SETUP ② — 나만의 도메인 연결 (구글 도메인 → Firebase 호스팅)

> 기본 주소 `https...web.app` 대신 `https://yeongjak.우리도메인.com` 같은 **나만의 주소**로 바꾸는 방법입니다.
> 도메인이 꼭 필요한 건 아니에요(web.app 주소로도 잘 동작). 보기 좋고 기억하기 쉬운 주소를 원할 때만 하세요.

먼저 **SETUP_Firebase.md 6단계까지(웹앱 배포)** 를 끝낸 상태여야 합니다.

---

## 0단계 — 도메인이 있나요?

- **있음:** 이미 산 도메인(예: 가비아·후이즈·Cloudflare·Squarespace Domains 등)이 있으면 그걸 쓰면 됩니다. → 2단계로.
- **없음:** 먼저 도메인을 하나 구입합니다. → 1단계.

> 참고: 예전의 "Google Domains" 서비스는 **Squarespace Domains** 로 이관됐습니다. 그래도 "구글 계정으로 산 도메인"이든 다른 등록업체든, 아래 방식(**DNS 레코드 2개 추가**)은 어디서나 똑같습니다.

---

## 1단계 — (도메인이 없다면) 구입

아무 도메인 등록업체에서 원하는 주소를 구입합니다.
- 한국: 가비아(gabia), 후이즈(whois), 카페24 등
- 해외: Cloudflare Registrar, Squarespace Domains, Namecheap 등

`우리학교이름.com` 처럼 1년 단위로 구매합니다(보통 1~2만 원/년).

---

## 2단계 — Firebase에 "커스텀 도메인 추가" 시작

1. Firebase 콘솔 → **빌드 > Hosting**.
2. 화면에서 **맞춤 도메인 추가**(Add custom domain) 클릭.
3. 쓰고 싶은 주소 입력. 두 가지 형태:
   - 서브도메인: `yeongjak.우리도메인.com`  ← **권장**(설정이 가장 단순, 보통 CNAME 1개)
   - 루트도메인: `우리도메인.com`        ← A 레코드 2개 필요
4. **계속** 누르면 Firebase가 **"이 레코드를 너희 도메인에 추가해"** 하고 값을 보여줍니다. 이 창을 **열어둔 채로** 3단계로 갑니다.

---

## 3단계 — 등록업체(DNS)에 레코드 추가

Firebase가 알려주는 값은 보통 둘 중 하나입니다.

### (A) 서브도메인을 골랐을 때 — CNAME 1개
도메인 등록업체의 **DNS 관리** 화면에서:

| 유형(Type) | 호스트/이름(Name) | 값(Value/Target) |
|---|---|---|
| CNAME | `yeongjak` | (Firebase가 보여주는 `...web.app` 같은 대상 주소) |

### (B) 루트도메인을 골랐을 때 — A 레코드 2개
| 유형 | 호스트/이름 | 값 |
|---|---|---|
| A | `@` (또는 비워둠) | (Firebase가 보여주는 첫 번째 IP) |
| A | `@` | (Firebase가 보여주는 두 번째 IP) |

> "호스트/이름"에 `@` 는 "도메인 자체(루트)"를 뜻합니다. 등록업체마다 칸 이름이 조금씩 달라요(이름/호스트/Host/서브도메인). 값만 정확히 넣으면 됩니다.

추가 후 저장합니다.

---

## 4단계 — 인증 & 연결 완료 기다리기

1. Firebase 창으로 돌아와 **확인/완료** 를 누릅니다.
2. Firebase가 소유권을 확인하고 **무료 SSL 인증서(https)** 를 자동 발급합니다.
3. 상태가 **"연결됨(Connected)"** 으로 바뀌면 끝.

> 시간: DNS는 보통 **몇 분~몇 시간**, 길면 최대 48시간까지 걸릴 수 있습니다(정상). 상태가 "대기 중(pending)"이어도 기다리면 됩니다.

연결되면 `https://yeongjak.우리도메인.com` 으로 접속됩니다. 🎉

---

## 5단계 — (권장) 함수 CORS를 내 도메인으로 좁히기

보안을 한 단계 더 올리려면, AI 함수가 **우리 도메인 요청만** 받도록 제한합니다.

1. `functions/index.js` 에서:
   ```js
   const ALLOW_ORIGIN = "*";
   ```
   를 실제 주소로 변경:
   ```js
   const ALLOW_ORIGIN = "https://yeongjak.우리도메인.com";
   ```
2. 재배포:
   ```bash
   firebase deploy --only functions
   ```

> 도메인을 여러 개(web.app + 커스텀) 쓰면 `*` 로 두는 게 편합니다. 운영이 안정되면 좁히는 걸 권장.

---

## 자주 막히는 곳

- "도메인 확인 대기 중"에서 안 넘어가요 → DNS 전파 대기(최대 48h). 레코드 값 오타 확인 후 기다리기.
- https 자물쇠가 안 떠요 → SSL 자동 발급에 시간이 걸립니다. 연결 직후엔 정상.
- 등록업체에 CNAME 칸이 없어요 → 대개 "DNS 레코드/레코드 관리/고급 DNS" 메뉴 안에 있습니다.
- 루트도메인에 CNAME이 안 돼요 → 루트(@)는 A 레코드를 쓰는 게 표준입니다(위 B표).
